# Тесты

Единственный санкционированный способ запускать `tests/*.test.sh` и `tests/*.test.mjs` - `tests/run`. Прямой запуск в обход него (`./tests/foo.test.sh`, `bash tests/foo.test.sh`, `node tests/foo.test.mjs`) упирается в отказ пролога (см. ниже), а для файлов без пролога резолвил бы боевой корень (`lib/runtime-root.sh`) и мог бы дотянуться до настоящего `systemctl`/`systemd-run`/`tmux`.

## Как запускать

```
tests/run                           # все обнаруженные тесты (кроме карантина)
tests/run foo.test.sh bar.test.sh   # только эти файлы (голое имя или tests/foo.test.sh)
tests/run --list                    # что обнаружено + карантин с причиной, без запуска
tests/run --help
```

## Почему нельзя напрямую

У тестового прогона и боевого рантайма (флот автономных воркеров под `systemd --user`, общий unit-шаблон `claude-auto@.service`) раньше не было механической границы - только "не забудь выставить маркер" на честном слове. Инцидент 20.07: прогон теста поднял посторонний systemd-юнит и оставил испорченную переменную в общем шаблоне. С тех пор граница - не пожелание, а три рубежа:

1. **Раннер** (`tests/run`) готовит песочницу до старта каждого теста.
2. **Пролог** (`tests/lib/bootstrap.sh` / `.mjs`) отказывает, если песочницы нет.
3. **Лит** (`tests/lint-bootstrap.test.sh`) отказывает, если новый тест забыл подключить пролог.

## Что делает раннер на каждый тест

- Временный корень (`mktemp -d`) + sentinel-файл `.claude-control-test-root` внутри него.
- Экспортирует `CLAUDE_CONTROL_TEST_ROOT=<временный корень>` - под этим маркером `lib/runtime-root.sh` резолвит корень (`CONTROL_DIR`/`DEPT_HOME`) только внутрь него; любая двусмысленность (не резолвится, пересекается с боевым каталогом, легаси-переменная `CLAUDE_CONTROL_DIR`/`CLAUDE_AUTO_HOME`/`DEPT_HOME` течёт наружу) - явный отказ, не тихий фолбэк на боевой путь.
- Заглушки `systemctl`/`systemd-run`/`tmux` внутри test root вместо настоящих бинарей - `lib/process-control.sh` требует их под маркером (для systemd unit-файлов, tmux-сокетов и т.п.).
- Уборка песочницы после теста (best-effort - переживает большинство прерываний, но не гарантия против SIGKILL).

**Ключевое свойство:** без маркера `CLAUDE_CONTROL_TEST_ROOT` поведение всех скриптов (bash и node) побитово прежнее. Граница включается только внутри тестов - боевой контур не меняется вообще.

## Обязательный пролог

Каждый `tests/*.test.sh` первой значимой строкой (шебанг / `set ...` / пустые строки / комментарии перед ней допустимы) подключает `tests/lib/bootstrap.sh`; каждый `*.test.mjs` - тем же образом `tests/lib/bootstrap.mjs`. Пролог переиспользует `lib/runtime-root.sh` и `lib/process-control.sh` (не дублирует их логику) и отказывает до первого побочного эффекта теста, если тест запущен в обход раннера.

`tests/lint-bootstrap.test.sh` проверяет это для всех файлов в `tests/` - новый тест без пролога роняет лит, а не проходит тихо.

## Карантин

Раннер физически не может исполнить некоторые тесты даже по явному имени - сейчас единственный такой файл: `tests/dept-exec-runner.test.sh` (e2e-тест, требует живую systemd user session - `dept-dispatcher` стартует раннер настоящим `systemd-run --user`, под заглушкой смысл теста исчезает). Карантинные тесты запускаются вручную на живом контуре, когда результат реально нужен; `tests/run --list` показывает список карантина с причиной.
