## Чек-ин при возврате (что реально может сдвинуться само)

Два сигнала - остальные пункты чисто триггерные и сами за неделю не изменятся.

1. **watchdog.log** - проверить, заработал ли мягкий heartbeat:
   ```
   grep -E 'miss [0-9]+/|kick' ~/.claude-control/watchdog.log
   ```
   - Есть `miss 1/2 - no kick yet` БЕЗ последующего `kick` -> мягкий heartbeat реально гасит ложные срабатывания (подтверждение пользы 2.1).
   - Снова пошли `kick` каждые ~5 минут -> либо вернуться к строгому heartbeat (п.1 ниже), либо копать причину (403/flap - это уже не watchdog).
   - История на 2026-05-31: последний `kick` был 2026-05-30 03:04 (старый watchdog, до правок); с тех пор тишина, мягкий heartbeat ни разу не сталкивался с промахом.

2. **Машинный API состояния**:
   ```
   claude daemon remote-control list --json
   ```
   - Если перестал отвечать `'daemon remote-control' is not available in this environment` -> открывается переход на структурный сигнал (пункты 5-6).

## Открытые пункты

### Отложено "по необходимости" (триггерные)
1. **Строгий heartbeat (полный 2.1).** Триггер: ложные `kick` у мягкого heartbeat. Дизайн - оставаться в мягком русле (больше окно / strip ANSI / N промахов), НЕ жесткий AND по healthy-маркеру (Codex предупреждал, что жесткий может ухудшить - лишние рестарты). Файл: `bin/claude-control-watchdog`.

### Сознательно вне объема (приняли как ограничение)
2. **Экзотические пути на tmux < 3.2.** newline/control chars в пути не гарантированы (фолбэк на `printf %q`, зависит от shell внутри tmux). На tmux >= 3.2 закрыто через `tmux -e`. Триггер: реально появятся такие пути на старом tmux. Файлы: `bin/claude-rc`, `bin/claude-control-run`.
3. **Атомарная ротация логов.** `cat`-over-inode может потерять/перемешать строки между `tail` и перезаписью при активной записи. Оставлено best-effort (тримим только по порогу размера). Триггер: замечена реальная потеря строк. Файл: `bin/claude-control-logrotate`.
4. **Traversal-check как sandbox.** Сейчас guard от опечаток, без `realpath`/резолва symlink (symlink на легитимную папку - валидный кейс). По модели "projects.yaml - доверенный файл". Трогать только при смене модели доверия. Файл: `bin/claude-rc`.

### Будущий spike (зависит от внешнего API)
5. **Машиночитаемый источник состояния.** `claude daemon remote-control list --json` пока недоступен публично (проверено локально 2026-05-30, `claude` 2.1.156). Если станет доступен - перевести health/status на него. См. чек-ин п.2 и `второе-мнение-2026-05-30-codex.md`.
6. **Замена парсинга TUI на структурный протокол.** Главный архитектурный долг по исходному аудиту. Упирается в п.5. Не первый шаг; делать, когда появится публичный сигнал состояния.

## Как продолжить
- Реализация: новая ветка от `main`, цикл "правка -> `bash -n`/`shellcheck` -> функциональный тест -> Codex-аудит -> коммит" (как в фазах 1-2).
- Установка `--link` (как сейчас у пользователя): правки в `bin/` живые сразу - симлинки `~/.local/bin/*` ведут на репо. Для юнитов нужен `./install.sh --link` (перезапускает control-сессию).
